# 모바일 웹 - 제페토 월드 플랫폼

> **프로젝트 코드네임**: ZEPETO Web World Platform (ZWWP)
> **문서 버전**: v0.1 (MVP)
> **최종 수정일**: 2026-04-27
> **문서 목적**: 본 문서는 Claude Code가 코드베이스를 생성/수정할 때 컨텍스트로 사용할 PRD입니다.

## 1. 프로젝트 개요 (Overview)

### 1.1 한 줄 요약
바이브 코딩(AI 보조 코딩)으로 제작된 HTML5 미니게임/레트로 게임을 모바일 웹 환경에서 업로드·배포·플레이할 수 있는 제페토의 신규 월드 플랫폼.

### 1.2 배경 (Why)
- 기존 제페토 월드는 **Unity + ZEPETO Script(TypeScript) + ZEPETO Studio 빌드 업로드** 파이프라인을 통해서만 제작 가능 → 진입장벽이 높음 (기존 Unity 개발자에게도 어려움).
- 최근 AI 기반 바이브 코딩(Claude Code, Cursor, Lovable 등)으로 일반 사용자도 **HTML/JS 기반 미니게임을 단시간에 제작 가능**해짐.
- 팩맨, 슈퍼마리오 클론, 플래피 버드류의 캐주얼/레트로 미니게임은 모바일 웹에서 충분히 동작함.
- 이 흐름을 제페토 생태계 안으로 흡수하여, **창작 진입장벽을 낮추고 월드 콘텐츠 공급량을 폭발적으로 늘리는 것**이 목표.

### 1.3 핵심 가치 제안
| 대상 | 제공 가치 |
|------|-----------|
| 크리에이터 | Unity·ZEPETO Studio 없이 HTML/JS 번들만 업로드하면 게임 출시 가능 |
| 플레이어 | 모바일 웹에서 즉시 미니게임 플레이, 친구에게 링크만 전달하면 친구와 함께 플레이 가능. 더 많은 게임을 플레이하려면 회원가입 유도 |
| 제페토 (플랫폼) | 창작자 풀 확대 + 콘텐츠 다양화 + 월드 카테고리 신규 축 확보 |

### 1.4 1차 (MVP) 범위
✅ **포함**
- 게임 리스트 페이지 (For You / Top Chart / Now Trending)
- 게임 상세 / 플레이 페이지 (iframe sandbox 기반 실행)
- 크리에이터 게임 업로드 (HTML/JS 번들 zip)
- 기본 메타데이터 관리 (제목, 썸네일, 설명, 카테고리)
- 좋아요 / 플레이 카운트 집계

❌ **제외 (후속 단계)**
- 수익화 (ZEM/Coin 결제, 광고)
- 라이브 룸 / 소셜 기능
- 댓글, 팔로우, DM
- 게임 자동 심사/AI 검수 (MVP는 운영자 수동 검수)
- URL 등록 방식 (외부 호스팅) — MVP는 파일 업로드만

## 2. 타겟 사용자 (Personas)

### 2.1 크리에이터 (Creator)
- **누구**: 바이브 코딩으로 HTML5 게임을 만든 일반 사용자, 인디 게임 개발자, 학생 개발자
- **니즈**: 만든 게임을 어딘가에 올리고 싶고, 트래픽이 보장된 플랫폼이면 좋겠음
- **기술 수준**: HTML/JS는 알지만 Unity/배포 인프라는 모를 수 있음
- **여정**: 회원가입 → 게임 zip 업로드 → 메타데이터 입력 → 검수 대기 → 출시 → 통계 확인

### 2.2 플레이어 (Player)
- **누구**: 제페토 모바일 앱 유저, 모바일 웹으로 캐주얼 게임을 즐기려는 유저
- **니즈**: 설치 없이 빠르게 즐길 수 있는 미니게임, 친구한테 공유 가능한 콘텐츠
- **여정**: 리스트 진입 → 썸네일/지표 보고 게임 선택 → 플레이 → 좋아요/공유

## 3. 정보 구조 (IA) & 화면 정의

### 3.1 사이트맵
```
/
├── /                       (홈 = 게임 리스트, For You 탭 기본)
│   ├── ?tab=for-you
│   ├── ?tab=top-chart
│   └── ?tab=now-trending
├── /games/:gameId          (게임 상세 + 플레이어)
├── /upload                 (크리에이터 업로드, 로그인 필요)
├── /studio                 (크리에이터 대시보드, 로그인 필요)
│   └── /studio/games/:id   (개별 게임 통계/수정)
├── /login
└── /profile/:userId        (간단 프로필)
```

### 3.2 핵심 화면 명세

#### (A) 홈 / 게임 리스트
**참조**: 첨부된 제페토 앱 월드 탭 UI와 동일한 정보 밀도/카드 레이아웃을 모바일 웹에서 재현.

- 상단 탭: `For You` / `Top Chart` / `Now Trending`
- 섹션 구조 (For You 기준):
  - `요즘 뜨는 게임 🔥` (가로 스크롤 카드)
  - `추천 미니게임` (가로 스크롤 카드)
  - `레트로 게임 모음` (가로 스크롤 카드)
- 카드 요소: 정사각형 썸네일 / 제목 / 좋아요% / 누적 플레이 수 / NEW·UPDATE 뱃지
- 하단 탭바: `Home` / `Live(비활성, 향후)` / `Create` / `Feed(비활성, 향후)` / `World` / `Profile`
  - MVP에서는 `Home`, `Create`, `Profile`만 활성화

#### (B) 게임 상세 / 플레이
- 상단: 썸네일 → 제목 → 크리에이터 → 좋아요/플레이 지표
- 메인: `▶ 플레이` 버튼 → 클릭 시 iframe 풀스크린 실행
- iframe은 `sandbox="allow-scripts allow-pointer-lock allow-orientation-lock"` 적용 (네트워크/스토리지 차단 또는 제한)
- 하단: 설명, 태그, 신고 버튼, 유사 게임 추천

#### (C) 업로드 화면 (크리에이터)
- Step 1: 게임 번들 업로드 (zip, 진입점 `index.html` 필수)
- Step 2: 메타데이터 입력 (제목, 한 줄 설명, 썸네일, 카테고리, 태그)
- Step 3: 사전 검증 (자동 정적 분석 → 통과 시 검수 큐로 이동)
- 결과: 검수 대기 상태 카드가 `/studio`에 표시

#### (D) 크리에이터 스튜디오
- 내가 올린 게임 목록 (상태: 검수중 / 출시 / 반려)
- 각 게임의 일별 플레이 수, 좋아요, 플레이 시간 평균

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 게임 디스커버리
- [F-DSC-01] 홈에서 For You/Top Chart/Now Trending 3개 탭 제공
- [F-DSC-02] For You는 단순 인기순 + 최신순 가중치 (MVP는 인기순 정렬, 추천 알고리즘은 후속)
- [F-DSC-03] Top Chart는 누적 플레이 수 기준
- [F-DSC-04] Now Trending은 최근 7일 플레이 증가율 기준
- [F-DSC-05] 게임 카드 클릭 시 상세 페이지 이동

### 4.2 게임 플레이
- [F-PLY-01] 게임은 iframe 안에서 실행됨
- [F-PLY-02] 모바일 가로/세로 모드 모두 대응 (게임 자체 설정 따름)
- [F-PLY-03] 플레이 시작/종료 시 이벤트 로깅 (플레이 카운트 +1, 세션 시간)
- [F-PLY-04] 좋아요 토글 (로그인 필요)
- [F-PLY-05] 신고 기능 (사유 선택 → 운영자 큐로 이동)
- [F-PLY-06] 공유 기능: 게임 상세/플레이 화면에서 공유 가능한 링크 제공 (비로그인 유저도 링크로 진입해 즉시 플레이 가능 — 실시간 동기화 없음, 각자 자기 화면에서 플레이)
- [F-PLY-07] 비로그인 플레이 카운트 제한: 일정 횟수(예: 3회) 이후에는 회원가입 유도 모달 표시 → "더 많은 게임을 플레이하려면 회원가입"

### 4.3 게임 업로드
- [F-UPL-01] zip 업로드 (최대 50MB, MVP 기준)
- [F-UPL-02] 진입점 `index.html` 필수, 없으면 거절
- [F-UPL-03] 외부 네트워크 호출 정적 분석 → 화이트리스트 외 차단 권고
- [F-UPL-04] 자동 검증 통과 시 운영자 검수 큐 진입
- [F-UPL-05] 운영자 승인/반려 → 크리에이터에게 알림

### 4.4 계정 / 인증
- [F-ATH-01] 이메일 + 소셜 로그인(카카오, 구글) — MVP는 카카오 1개 + 이메일
- [F-ATH-02] 향후 제페토 계정과의 OAuth 연동 인터페이스만 분리해둠 (실제 연동은 후속)

### 4.5 운영자 콘솔 (어드민)
- [F-ADM-01] 검수 큐 관리 (게임 미리보기 + 승인/반려)
- [F-ADM-02] 신고 큐 관리
- [F-ADM-03] 게임 강제 비공개 처리

## 5. 비기능 요구사항 (Non-Functional Requirements)

### 5.1 성능
- 리스트 페이지 LCP < 2.5초 (3G Fast 기준)
- 게임 iframe 로드는 게임 자체 사이즈에 의존 (50MB 기준 기대치 제공)
- API p95 응답 < 300ms

### 5.2 보안 / 샌드박싱 ⚠️ **최우선**
- 업로드된 게임은 반드시 **별도 도메인**에서 서빙 (예: `play.zwwp.example.com`) → 메인 도메인 쿠키 격리
- iframe `sandbox` 속성 강제, CSP 헤더로 fetch 도메인 제한
- zip 해제 시 `path traversal` 방어 (절대경로/`..` 거부)
- 업로드 파일에 대한 정적 분석:
  - `eval`, `Function()`, 외부 스크립트 로드 패턴 검출 → 경고
  - 알려진 악성 페이로드 시그니처 매칭
- XSS, CSRF 표준 대응

### 5.3 확장성
- 트래픽 증가 시 게임 정적 자산은 CDN으로 서빙
- 메타데이터 DB와 게임 번들 스토리지 분리

### 5.4 접근성 / 호환성
- 우선 지원: iOS Safari 16+, Android Chrome 최신 2버전
- 데스크톱은 모바일 뷰포트 시뮬레이션으로 동작 (반응형은 MVP 범위 외)

| 리스크 | 영향 | 대응 |
|--------|------|------|
